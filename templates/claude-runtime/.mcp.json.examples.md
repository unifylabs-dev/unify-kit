# `.mcp.json` worked examples

The shipped `.mcp.json.template` is intentionally minimal — `{ "mcpServers": {} }`.
Pick the servers your stack actually uses, paste the JSON block into the
`mcpServers` object, then set the referenced environment variables (in your
shell, in `.env.local`, or via your secret manager). The Claude Code MCP loader
expands `${VAR}` against your shell environment at session start.

## Supabase

For a project using Supabase Postgres + Auth + Storage:

```json
{
  "supabase": {
    "command": "npx",
    "args": ["-y", "@supabase/mcp-server-supabase@latest"],
    "env": {
      "SUPABASE_URL": "${SUPABASE_URL}",
      "SUPABASE_ANON_KEY": "${SUPABASE_ANON_KEY}",
      "SUPABASE_SERVICE_ROLE_KEY": "${SUPABASE_SERVICE_ROLE_KEY}"
    }
  }
}
```

`SERVICE_ROLE_KEY` is a powerful key. Scope it via Supabase RLS + grant it to
the MCP server only for sessions where read/write to your project DB is the
intent. Leave it out for read-only workflows.

## Playwright

For projects with Playwright e2e tests where you want Claude to drive the
browser during a session:

```json
{
  "playwright": {
    "command": "npx",
    "args": ["@playwright/mcp@latest"]
  }
}
```

No env vars required. Useful for verification flows where Claude opens the
running dev server in a real browser and asserts the rendered DOM.

## context7

For pulling current library docs into a session (replaces stale training-data
recall for API surfaces that change between versions):

```json
{
  "context7": {
    "command": "npx",
    "args": ["-y", "@context7/mcp-server@latest"]
  }
}
```

No env vars required. Useful when working with libraries that ship breaking
changes between minor versions (Prisma, Next.js, Tailwind, etc.).

## How env vars flow

- The MCP loader substitutes `${VAR_NAME}` from the shell environment at
  session start. Set them in your shell rc-file, in `.env.local` loaded by
  your dev tooling, or via your editor's run-config.
- Don't commit secrets. `.env.local` is gitignored by default in most
  scaffolds; verify yours is.
- Per-project `.mcp.json` lives in the repo and IS committed. Only the
  *references* to env vars live here; the values are local.

## Adding more servers

Run `/plugin marketplace list` and `/plugin install <server>` inside Claude
Code for any MCP server published as a plugin. For ad-hoc servers, follow the
upstream README and add a block to `mcpServers` here.
